# Vendored OpenAPI schema

`schema.yaml` in this directory is a **checked-in copy** of the Django
backend's published OpenAPI contract (generated by drf-spectacular).

- Source of truth: `api/schema.yaml` in the `ITSE500-Django` repository.
- This copy is what `swagger_dart_code_generator` reads to emit the typed
  Dart client under `lib/api/generated/` (see `build.yaml` at the repo
  root).
- **Do not hand-edit this file.** When the backend contract changes,
  re-copy `api/schema.yaml` from the Django repo over this file, then run:

  ```
  flutter pub get
  dart run build_runner build --delete-conflicting-outputs
  ```

  and commit both the updated schema and the regenerated client together.
- CI enforces this: a drift-check job re-generates the client from this
  vendored schema and fails if `lib/api/generated` differs from what's
  committed, so a stale generated client can never merge silently.
